## Failed Approaches — Don't Repeat These  ⚠️ MANDATORY

- **Tentato:** `Path("templates") / "index.html"` (path relativo) nella route Flask `/`
  **Motivo fallimento:** il CWD del processo era `/mnt/backup/PROGETTI/tavolarotonda` ma il template risiedeva in `gui/templates/`, e `Path("templates")` risolve al CWD. Risultato: `FileNotFoundError` 500 sul primo avvio.
  **Approccio attuale:** usare `ROOT / "gui" / "templates" / "index.html"` con `ROOT = Path(__file__).parent.parent` — sempre canonico, indipendente da CWD.

- **Tentato:** Solo cache-bust del CSS senza fix server Flask
  **Motivo fallimento:** Chrome mantiene cache aggressiva anche su `?v=xxx` se l'header `Cache-Control` non è esplicito.
  **Approccio attuale:** doppio intervento — Flask serve `Cache-Control: no-cache, must-revalidate` + `ETag` per static, `Cache-Control: no-store` per index, E `?v=<sha>` come belt-and-suspenders.

- **Tentato:** Lasciare il vecchio `payload.audit_target = audit` DOPO il blocco `if (state.mode === "audit") { ... }` durante il refactor
  **Motivo fallimento:** riga duplicata 352, sovrascrive il payload esteso perdendo `audit_project_mode` e `audit_exclude`.
  **Approccio attuale:** singolo blocco `if (state.mode === "audit")` che setta tutte e 3 le proprietà del payload.

- **Tentato:** OCR interpretava il placeholder "Pronto per iniziare" come "Stream del Dibattito tagliato"
  **Motivo fallimento:** AI tool ha confuso il messaggio statico con contenuto dinamico mancante.
  **Approccio attuale:** placeholder reso inequivocabile con bordo tratteggiato 2px + header "Pronto per iniziare" + CTA "▶ Premi Avvia Tavola Rotonda nella sidebar →" + 3 mockup events.

## Key Decisions

| Decisione | Rationale | Alternative scartate |
|---|---|---|
| Modalità progetto = path è directory (toggle ON di default) | Auto-detect se path finisce con `/` o è directory esistente | Tree picker UI (più complesso, ~2x codice) |
| `AUDIT_INCLUDE_EXTS` = whitelist 30+ ext | Permette documentazione + config + codice; blocca binari | Glob pattern regex (più fragile) |
| `AUDIT_EXCLUDE_DIRS` = skip ricorsivo hardcoded | Anti-DoS automatico (no `.git`, no `.venv`) | Allowlist (troppo permissiva, rischio esposizione) |
| Limiti: 50 file × 4000 chars × 200KB | Sufficiente per ~95% dei progetti reali, protegge contesto LLM | Nessun limite (rischio DoS / context overflow) |
| Tree ASCII invece di JSON nested | Più leggibile in output LLM, cap 80 righe | JSON (rumoroso, verbose) |
| Compatibile con audit singolo file (legacy invariato) | Backward-compatible con utenti che passano un singolo `.py` | Solo progetto (breaking change) |
| Cache-bust con `?v=<git_sha>` | Git SHA sempre disponibile, auto-incrementale | Timestamp (cambia ad ogni render, no-cache troppo aggressivo) |
| Path canonici Flask = `ROOT / "gui" / "..."` | MAI path relativi (bug visto) | `os.path.dirname(__file__)` (funziona ma meno idiomatico) |

## Edge Cases & Error Handling

- **Path non esiste** (singolo file o directory) → `raise FileNotFoundError(f"path non trovato: {audit_target}")` → Flask ritorna 500 con messaggio esplicito.
- **Directory vuota o nessun file leggibile** → `raise ValueError(f"nessun file leggibile trovato in {target_path}")`.
- **File binario** (.png, .jpg) → skip automatico per `_is_auditable()` (estensione non in whitelist).
- **Permission denied su un file** → `try/except (PermissionError, OSError)` → skip silenzioso.
- **Troppi file** (>50) → tronca a 50, gli altri scartati (marcato nel topic come "N file ma analizzo solo i primi 50").
- **File singolo molto grande** (>4000 chars) → tronca con marker `... [truncated at 4000 chars, file totale più lungo]`.
- **Totale sforato** (>200KB) → tronca con marker `... [truncated, audit budget esaurito]`.
- **Toggle modalità progetto OFF + path directory** → fallback al comportamento legacy (`target_path.read_text()` come se fosse file singolo, fallirà se è dir).

## Warnings / Gotchas

- **Path canonici Flask:** SEMPRE `ROOT / "gui" / "..."` nei route handler, MAI `Path("...")` relativo (bug già visto).
- **Cache-bust:** se committi codice ma il server è già avviato col vecchio build_id, devi restartare Flask (`pkill -9 -f "gui/app.py" && cd <proj> && python3 gui/app.py ...`). Flask legge `_get_build_id()` solo all'avvio.
- **Server cwd:** se vedi 404 su `/static/style.css?v=xxx` con cache vecchia, verifica `ls -la /proc/$(pgrep -f 'gui/app.py')/cwd` → se è `/tmp/tavolarotonda` stai servendo la copia vecchia. Kill e restart dal path canonico.
- **auto-memory vs vault:** preferenze utente e quick-fact vanno in `~/.claude/.../memory/`, NON nel vault. Il vault è per conoscenza durevole (decisioni, root-cause, concetti, sessioni).
- **Multi-subagent:** la skill `wiki` vieta ai subagent di spawnare sub-subagent. La modalità delegata (lever B) di `/wiki all` la fa SEMPRE il main (Opus).
- **Doppio commit failed_approach in git:** non committare `.env`, secrets, `__pycache__/`, `node_modules/`. Il `.gitignore` dovrebbe già coprirli.
